# 📊 Customer Churn Prediction

## 🎯 Objective
The goal of this project is to predict whether a customer will churn (leave the service) or not using machine learning techniques.

---

## 📁 Dataset
- Telco Customer Churn Dataset
- Contains customer details like tenure, charges, services, etc.

---

## 🛠️ Tools & Technologies
- Python
- Pandas
- NumPy
- Scikit-learn
- Matplotlib / Seaborn

---

## ⚙️ Data Preprocessing
- Handled missing values
- Converted TotalCharges to numeric
- Removed unnecessary column (customerID)
- Applied one-hot encoding on categorical variables

---

## 🤖 Model Building
- Logistic Regression
- Train-test split used for validation

---

## 📈 Model Performance
- Accuracy: ~80%
- Good performance on non-churn customers
- Lower recall for churn class due to class imbalance

---

## 📊 Evaluation Metrics
- Accuracy
- Precision
- Recall
- F1-score
- Confusion Matrix

---

## 💡 Key Insights
- Model predicts non-churn customers effectively
- Churn prediction needs improvement
- Class imbalance impacts performance

---

## 🚀 Future Improvements
- Use Random Forest / XGBoost
- Apply SMOTE for class balancing
- Hyperparameter tuning
